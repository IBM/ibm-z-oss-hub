Container Images for IBM Z

This is a channel of open source container images available for use, free of charge, that can be pulled and managed through the common graphical and command line interfaces that support containerized workloads across platforms.

These images run in both the Linux, and zCX environments of z/OS on IBM Z. All of these images are OCI-compliant, and built for deployment in a local environment. If you are looking for cloud-enabled containers for IBM Z, you may want to visit the Red Hat Marketplace.

Several steps have been taken to ensure the veracity of these container images:

  • The images are built from source - they are not assembled from images hosted at other registries.
  • Build tests from the originating project are run when the container image is built.
  • Container images are scanned for known vulnerabilities and the reports are provided at the image's associated web page.
  • The hash for each container image is published. This enables users to avoid installing an inintended image version.

Getting Started

This collection of container images is hosted at the IBM Cloud Container Registry (icr.io). An IBM Cloud ID is required to pull images from the registry. Any type of IBM Cloud ID will work, including "lite" IDs with no associated cost. You are welcome to browse the set of available images here, but the ID is required at docker/podman login time.

Each container image has a home page with details about container use, a vulnerability report, and a docker pull string. It is strongly recommended that you pull images using the sha256 hashes provided to ensure that you get the container image instance you are requesting.

Please see the demonstration video in this section for an example of how to request an IBM cloud ID, and use it from the docker command line to login and pull a cointainer image.
